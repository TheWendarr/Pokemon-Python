# Roadmap

Each phase has acceptance criteria; a phase isn't done until they pass.

## Phase 0 — Foundations  [DONE]
Repo structure, canonical data models, GameData repository, PokeAPI
pipeline (REST + CSV) generating Gen 1-5 data with Gen 5 values.
*Accepted:* dataset generated and value-checked (Thunderbolt 95, no
Fairy, Steel resists Ghost/Dark).

## Phase 1 — Battle core  [DONE]
Pure seeded engine: Gen 5 damage, statuses, stages, ordering, PP +
Struggle, switching, items, flee/catch, baseline AIs, CLI demo.
*Accepted:* 50 pytest cases incl. hand-verified damage; seeded AI-vs-AI
battle runs to completion on real data.

## Phase 2 — Battle completeness  ✅ DONE
~35 abilities (Intimidate, Levitate, Static, Guts, Sturdy, weather
setters, pinch boosts, contact punishers, absorbers...), 11 held items
(Leftovers, berries, choice items, Life Orb, Focus Sash...), all four
weathers, entry hazards (Stealth Rock / Spikes / Toxic Spikes + Rapid
Spin), screens + Safeguard/Mist/Lucky Chant/Tailwind, Protect/Endure
with consecutive-use decay, two-turn / recharge / rampage moves,
partial trapping, Leech Seed, choice locks, forced switching, and ~50
special-case handlers (Counter, Encore, Taunt, Baton Pass, Curse, Pain
Split...). KO-aware GreedyAI.
*Accepted:* 91 pytest cases; EFFECT_SKIPPED rate 3.6% across 150 random
full-dex AI battles (`python -m pkmn.cli.coverage`), under the 5% bar.

## Phase 3 — Overworld v1  ✅ DONE
Pygame-CE client (`python -m pkmn.game.play`): scene stack, Tiled maps
via pytmx, grid movement with collision, NPCs (incl. a healer) and
signs with dialog boxes, town<->route warps, tall-grass encounters from
per-map tables, and a full battle UI (Fight/Bag/Pkmn/Run, party menu,
forced replacements, catching, whiteout) that renders the engine's
Event stream through the same formatter as the CLI demo. All art and
maps are generated by `tools/make_assets.py` — zero third-party assets.
*Accepted:* 9 headless client tests drive the real Game object frame by
frame: walk the town, warp to Route 1, trigger a grass encounter, win a
battle, catch a wild Pokemon into the party, white out, talk and heal.

## Phase 4 — Events & scripting  ✅ DONE
Flag store + money on GameState; triggers (step-on tiles, interact,
auto-on-map-enter, all gated by `unless_flag`); a JSON script command
set (say / heal / give_item / give_money / set_flag / clear_flag /
if_flag / warp / move_npc / face_npc / hide_npc / battle) interpreted
by `pkmn/game/script.py`, which yields to scenes and resumes when they
pop; trainer line-of-sight battles with the "!" pause, walk-over
cutscene, defeat flags, prize money, and post-defeat dialogue; plus the
first interior (Hexton clinic) behind a door warp.
*Accepted:* the scripted rival battle on Route 1 — pre-battle dialogue,
a Snivy fight, victory rewards, and post dialogue all gated by the
`beat_rival` flag — runs end to end in tests/test_scripting.py.

## Phase 5 — Game systems  ✅ DONE
Whole-game save/load (GameState -> JSON: party, PC box, bag, money,
flags, location; `--save` flag autoloads on launch); experience with
all six official growth curves, level-ups with stat recalculation,
level-up move learning, and post-battle evolution (data from the
pipeline's new `evolves_to` edges, REST and CSV); START pause menu with
party list (summary screen, order swapping), an overworld bag that
targets party members (heals, cures, revives), in-menu saving, and a
two-column PC box on the clinic terminal. Catches with a full party go
to the PC. The clinic doubles as the Pokemon Center.
*Accepted:* catch -> train -> evolve (Oshawott to Dewott mid-session)
-> save -> reload -> continue, all covered in tests/test_systems.py.

## Phase 6 — Authoring toolkit  ✅ DONE
"Game folder" format: a manifest (game.json: name, start, starter,
bag, money, flags, optional dex subset) plus maps/, tileset, sprites,
scripts.json, and encounters.json — fully asset-agnostic (the engine
reads every path from the folder; tilesets resolve from the TMX).
`python -m pkmn.cli.lint --game DIR` validates everything: manifest,
map parse + ground layers, warp targets/bounds/walkability, script
commands and their species/item/map references, trainer party specs,
encounter tables, dex-subset enforcement, and sprite presence. Hexton
itself is now just the reference game folder, and `examples/isleton`
is a second region (own palette, own sprites, totodile starter, a
sailor gatekeeper, and a shell-shrine goal) built only with the format.
See docs/AUTHORING.md.
Post-roadmap polish: manifest feature toggles + settings, per-map
weather and encounter rates, choice/shop/give_pokemon/money script
commands, held items in trainer specs, designer starters (moves or no
starter at all), move-replacement prompts, run button, species form
aliases, and the `examples/triad` showcase (3 cities / 3 themed routes
/ 8 unique trainer battles), all under tests/test_showcase.py.

*Accepted:* tests/test_authoring.py boots `examples/isleton` — a folder
verified to contain no .py files — and plays it start to finish:
manifest-driven new game, cross the island, forced-grass cove, Sailor
Brom's line-of-sight battle, and the flag-gated shrine ending.
